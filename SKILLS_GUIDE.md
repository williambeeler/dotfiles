# Codex Skills Guide

This file explains the skills in `config/codex/skills` in plain English.

You do not need to memorize these.
Think of a skill as a specialized playbook Codex can use when a request matches that topic.

## How to read this guide

- **What it means** explains the skill in normal language.
- **You might use it for** gives examples of real requests.

## General Product, Marketing, and Growth Skills

### `ab-test-setup`
**What it means:** Helps plan and structure an experiment so you can test whether one version performs better than another.
**You might use it for:** "Help me set up an A/B test for this pricing page headline."

### `analytics-tracking`
**What it means:** Helps decide what user actions to track and how to measure them in tools like GA4 or GTM.
**You might use it for:** "Set up event tracking for my signup flow."

### `competitor-alternatives`
**What it means:** Helps write pages that compare your product to a competitor, or position you as an alternative.
**You might use it for:** "Create a 'Why use us instead of Calendly?' landing page."

### `content-strategy`
**What it means:** Helps decide what content you should create and why.
**You might use it for:** "What blog topics should I focus on for my SaaS?"

### `copy-editing`
**What it means:** Improves copy you already wrote by making it clearer, stronger, and more polished.
**You might use it for:** "Tighten up this homepage copy."

### `copywriting`
**What it means:** Writes or rewrites marketing copy for pages, headlines, CTAs, and sections.
**You might use it for:** "Write better copy for my pricing page."

### `email-sequence`
**What it means:** Helps create a sequence of emails that work together, like onboarding, nurture, or re-engagement flows.
**You might use it for:** "Write a 5-email welcome sequence for new users."

### `free-tool-strategy`
**What it means:** Helps think through building a free tool as a marketing asset.
**You might use it for:** "What free tool could I build to attract leads?"

### `launch-strategy`
**What it means:** Helps plan how to launch a product, feature, or update.
**You might use it for:** "Help me launch this new feature next month."

### `marketing-ideas`
**What it means:** Brainstorming support for practical ways to grow a product.
**You might use it for:** "Give me 20 marketing ideas for my scheduling app."

### `marketing-psychology`
**What it means:** Applies persuasion, buyer behavior, and mental models to your messaging.
**You might use it for:** "How can I make this landing page more persuasive?"

### `paid-ads`
**What it means:** Helps with ad strategy, ad copy, targeting, and campaign structure.
**You might use it for:** "Help me draft LinkedIn ads for this product."

### `pricing-strategy`
**What it means:** Helps think through pricing, packages, plans, and monetization.
**You might use it for:** "Should I charge per seat or per usage?"

### `product-marketing-context`
**What it means:** Creates a shared background document about your product, audience, messaging, and positioning.
**You might use it for:** "Set up a reusable product marketing context for this app."

### `programmatic-seo`
**What it means:** Helps plan large groups of SEO pages built from templates and data.
**You might use it for:** "How could I create location pages at scale?"

### `referral-program`
**What it means:** Helps design or improve a referral or affiliate program.
**You might use it for:** "Help me create a refer-a-friend program."

### `schema-markup`
**What it means:** Helps add structured data so search engines understand your pages better.
**You might use it for:** "Add FAQ schema to this page."

### `seo-audit`
**What it means:** Reviews a site or page for SEO problems and missed opportunities.
**You might use it for:** "Why is this page not ranking?"

### `social-content`
**What it means:** Helps create social media posts and adapt them for different platforms.
**You might use it for:** "Turn this blog post into a LinkedIn post and an X thread."

### `write-freek-dev-blogpost`
**What it means:** A style-specific writing guide for posts that are meant to sound like freek.dev.
**You might use it for:** "Draft a blog post in Freek's usual style."

## Coding, Architecture, and Debugging Skills

**What it means:** Pulls in fresh library and framework documentation when coding questions depend on up-to-date docs.
**You might use it for:** "How do I configure middleware in the latest Next.js?"

### `fix-github-issue`
**What it means:** A workflow for taking a GitHub issue, fixing it in code, and preparing a PR.
**You might use it for:** "Fix issue #42 in this repo."

### `laravel-debugger`
**What it means:** Specialized help for diagnosing Laravel problems.
**You might use it for:** "This Laravel queue job keeps failing. Find the bug."

### `laravel-feature-builder`
**What it means:** Helps build a complete Laravel feature from requirement to implementation.
**You might use it for:** "Add a user role management feature in Laravel."

### `laravel-inertia-react-structure`
**What it means:** Guides how to organize frontend code in a Laravel + Inertia + React project.
**You might use it for:** "Where should these Inertia React components live?"

### `laravel-simplifier`
**What it means:** Refactors Laravel or PHP code so it is cleaner and easier to maintain without changing behavior.
**You might use it for:** "Simplify this controller without changing what it does."

### `pdf`
**What it means:** Helps read, generate, split, merge, and fill PDFs.
**You might use it for:** "Extract the text from this PDF" or "Fill in this PDF form."

### `php-guidelines-from-spatie`
**What it means:** A coding standards guide for writing Laravel and PHP in a way Spatie prefers.
**You might use it for:** "Make this PHP code match Spatie's conventions."

### `ray-skill`
**What it means:** Sends debug output to the Ray app so you can inspect data visually.
**You might use it for:** "Send this query result to Ray while debugging."

### `react-best-practices`
**What it means:** Helps write React code more cleanly, especially around hooks, state, and effects.
**You might use it for:** "Review this component for bad React patterns."

### `react-components`
**What it means:** Turns visual designs into React components in a structured way.
**You might use it for:** "Convert this design into reusable React components."

### `react-doctor`
**What it means:** A final check after React changes to catch likely mistakes.
**You might use it for:** "Review my React changes before I ship them."

### `react-native-best-practices`
**What it means:** Helps make React Native apps smoother and faster.
**You might use it for:** "Why is this React Native screen janky on scroll?"

### `react-state-management`
**What it means:** Helps choose and use state tools like Redux, Zustand, Jotai, or React Query.
**You might use it for:** "Should this app use Zustand or Redux Toolkit?"

### `review-pr`
**What it means:** A workflow for reviewing and merging pull requests, especially in a GitHub-heavy workflow.
**You might use it for:** "Review this PR and tell me if it's safe to merge."

### `spatie-package-skeleton`
**What it means:** Helps scaffold a new PHP or Laravel package using Spatie's preferred setup.
**You might use it for:** "Create a new Laravel package for this feature."

### `task-planner`
**What it means:** Breaks a big technical task into manageable steps.
**You might use it for:** "Plan the work to migrate this app from Vue to React."

### `update-spatie-docs`
**What it means:** Helps import or refresh docs for a Spatie package.
**You might use it for:** "Update the docs for the media-library package."

### `vercel-react-best-practices`
**What it means:** Focuses on React and Next.js performance patterns recommended by Vercel.
**You might use it for:** "Optimize this Next.js page for better performance."

## Quick Shortcuts By Situation

If you are not sure which skill fits, here are some easy starting points:

- **I want better marketing copy:** `copywriting`, `copy-editing`, `page-cro`
- **I want growth ideas:** `marketing-ideas`, `launch-strategy`, `free-tool-strategy`
- **I want SEO help:** `seo-audit`, `schema-markup`, `programmatic-seo`, `competitor-alternatives`
- **I want to improve signup or conversion:** `signup-flow-cro`, `form-cro`, `popup-cro`, `page-cro`
- **I need Laravel help:** `laravel-debugger`, `laravel-feature-builder`, `laravel-simplifier`, `php-guidelines-from-spatie`
- **I need React help:** `react-best-practices`, `react-doctor`, `react-state-management`, `vercel-react-best-practices`
- **I need design help:** `frontend-design`, `web-design-guidelines`, `building-native-ui`
- **I need GitHub workflow help:** `fix-github-issue`, `review-pr`, `convert-github-issue-to-discussion`

## A Practical Way To Use This

You can talk naturally.
You do not need to say the skill name exactly.

Examples:

- "Help me improve conversions on this signup page."
- "Review this React component for bad patterns."
- "Plan a launch for this feature."
- "Fix this Laravel error."
- "Draft a few LinkedIn posts from this blog article."

If your request clearly matches one of these areas, Codex can use the matching skill automatically.
